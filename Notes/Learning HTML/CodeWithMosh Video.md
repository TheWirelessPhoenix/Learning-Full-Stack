## Code With Mosh Learning HTML Notes

## Section One Web Architecture and Protocols

1. Client and Server Communication
Web applications rely on a two way communication system. The client, which is the web browser used by the person viewing the site, sends a request for data. The server, which is a remote computer storing the website files, processes this request and returns a response containing the necessary assets.

2. Hypertext Transfer Protocol and Security
Data travels across the internet using the Hypertext Transfer Protocol. When a website utilizes the secure version, an extra layer of encryption protects the transmitted information, ensuring privacy and data integrity for users.

## Section Two HTML Fundamentals

1. The Doctype Declaration
Every document must begin with a document type declaration. This instruction informs the web browser that the file is an HTML document, which ensures the browser renders the layout correctly rather than falling back to an outdated rendering mode.

2. Element Tags and Structure
HTML elements are defined by opening and closing tags. An opening tag marks the start of an element, while a closing tag marks the end. The closing tag is distinguished by a forward slash character inside the tag.

3. Void Elements
Certain structural items, such as the image element, are classified as void elements. These items do not contain nested text or other tags, meaning they do not require a closing tag to function properly.

4. Code Comments
Developers insert comments into the source code to document their work or leave notes for teammates. The web browser ignores these notes completely during the rendering process, ensuring they remain hidden from the public view.

## Section Three Head Element and Metadata

1. The Head Element Function
The head portion of a document acts as a repository for information about the page itself. Unlike the body element, the content placed within the head is not displayed directly to the visitors of the website.

2. Meta Descriptions
A meta description provides a concise summary of the webpage content. Search engines display this text snippet in search results to help users understand what the page is about before clicking the link.

## Section Four Document Object Model

1. Document Object Model Definition
The Document Object Model is a programming interface for web documents. It represents the structure of the webpage as a tree of nodes, enabling programming languages to dynamically access, alter, and update the style or content of the document.

## Section Five CSS Styling and Layout Basics

1. Rounded Images and Borders
To transform a square image into a perfect circle, developers adjust the border radius property. Setting this value to exactly half of the total pixel width or height of the image curves the corners completely.

2. Float Directions
The float property changes how elements sit on a page by pushing an item to a specific side. This allows surrounding text or adjacent elements to wrap naturally around the floated item.

3. Margin Spacing
Margins are used to generate space outside the boundaries of an element, pushing neighboring items away. Developers can specify exact spacing for individual directions such as margin top, margin bottom, margin left, or margin right.

4. Typography Styles
Font weight controls the thickness or boldness of text characters. Font styles allow developers to choose specific text variations, such as italicizing words for emphasis.

5. Element Classes and Naming
Classes are identifiers assigned to elements so developers can target multiple items simultaneously with consistent styles. Choosing descriptive naming conventions keeps the stylesheet clean and organized.

6. Percentage Units
Percentages are relative units of measurement used in responsive web design. They specify dimensions based on the size of the parent container, allowing layouts to scale smoothly on different screen sizes.

## Section Six Frameworks and Efficiency

1. Time Saving Frameworks
Frameworks provide prewritten code structures and reusable components. Utilizing these tools saves substantial development time because engineers do not have to build fundamental features from scratch.
